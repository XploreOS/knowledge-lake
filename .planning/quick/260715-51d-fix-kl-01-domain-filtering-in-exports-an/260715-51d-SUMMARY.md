# Quick Task 260715-51d — Summary

## What was done

Closed **KL-01** (domain-scoped exports were not domain-scoped — a 62%-foreign
export measured in the audit) and **KL-02** (LLM budget caps under-enforced by
2.4x-9x due to a proxy-alias/Bedrock-ID key mismatch).

### Task 1 — KL-01: filter rows by domain in all three exports

`export_rag_corpus`, `export_pretrain_corpus`, and `export_finetune_dataset`
in `src/knowledge_lake/pipeline/export.py` all accepted a `domain` kwarg but
only used it to build the S3 path segment / tag — the row query ignored it.
Each function now resolves the per-row source domain via
`registry_repo.get_domain_for_source()` and, when `domain is not None`,
excludes any row whose resolved domain does not match (strict equality —
unclassified rows are excluded from a domain-scoped export). `domain=None`
is unchanged: no filtering, `_unclassified` path, matching the pre-existing
CLI/API default exactly (locked decision). Each function now logs
`total`/`kept`/`filtered_out` (rag: `export.rag_corpus.building`, pretrain:
`export.pretrain.qualifying`, finetune: `export.finetune.building`) so an
operator can see scoping happened.

### Task 2 — KL-02: correct cost attribution

`compute_call_cost()` in `src/knowledge_lake/llm/pricing.py` now tries, in
order: (1) `response._hidden_params["response_cost"]` when present and > 0
— the LiteLLM proxy's own authoritative cost; (2)
`litellm.completion_cost(completion_response=response)`; (3) a flat
per-1k-token estimate from `settings.enrich.fallback_cost_per_1k_*`, now
logged at `warning` as `enrich.cost_calc_fallback_estimate` with an explicit
under-estimation note (previously a bare `enrich.cost_calc_failed` with no
guidance). `bootstrap_llm_pricing()` now registers under the alias names
actually echoed back by the proxy (`cheap_model`, `strong_model`,
`eval_model`) with `litellm_provider="openai"`, instead of Bedrock model IDs
that the proxy never returns in `response.model`. Added
`eval_model_bedrock_id`, `eval_model_input_cost_per_token`, and
`eval_model_output_cost_per_token` to `EnrichSettings` (mirroring
`strong_model`'s values — `eval_model` maps to the same Sonnet backend per
`infra/litellm/config.yaml`). Both functions retain the never-raise contract
(D-05).

### Task 3 — Tests for both fixes

- `tests/unit/test_export_domain_filter.py` (new, 6 tests): seeds sources
  classified `aviation`, `functional-medicine`, and unclassified (`None`),
  and asserts all three export functions return ONLY the requested domain's
  rows with `domain="aviation"`, plus a regression guard that `domain=None`
  still returns everything (the preserved default).
- `tests/unit/test_llm_pricing.py` (new, 10 tests): asserts the 3-level
  fallback chain in order, that a `response_cost == 0.0` is not trusted and
  falls through, that the flat-estimate path warns with an under-estimation
  note (not the old bare `cost_calc_failed`), that pricing never raises even
  with no `_hidden_params` and no `usage`, and that `bootstrap_llm_pricing`
  registers exactly the three alias keys under `litellm_provider="openai"`.

### Task 4 — Make filtering reachable from CLI and API

Added `--domain`/`-D` to `klake export` (all three export functions were
previously always called with no `domain`, making the KL-01 fix unreachable
except via Dagster) and an optional `domain` field (min_length=1,
max_length=100, mirroring `WikiExportRequest`'s pattern) to `ExportRequest`
in `src/knowledge_lake/api/schemas.py`, threaded through in
`src/knowledge_lake/api/app.py`'s `export_endpoint`. Default `None`
preserves existing behavior exactly. Regenerated `docs/openapi.json` via
`uv run klake openapi` (CI-verified, determinism confirmed by re-running
with no further diff).
