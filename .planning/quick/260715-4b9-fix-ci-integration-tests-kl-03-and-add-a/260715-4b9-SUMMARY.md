### Task 1 — `klake_test` database

Added a `klake_test OWNER klake` block to `infra/postgres/init.sql`, following
the exact idempotent pattern (`SELECT ... WHERE NOT EXISTS ... \gexec`, then
`\c` + GRANT, then `\c klake`) already used for `dagster_storage` /
`litellm_storage`. Schema is still Alembic's job — this only ensures the empty
database exists (FOUND-09). Since the running container's volume predates this
change, created `klake_test` manually against the live container to verify
locally, per the task's environment notes.

### Task 2 — Stale `put_raw` mocks

Added `domain=None, tags=None` to all six `mock_put_raw` signatures across
`test_upload.py`, `test_ingest_url_dedup.py`, and `test_dedup_noop.py` so they
match the production `StorageBackend.put_raw(source_id, data, ext, session,
mime_type=None, domain=None, tags=None)` signature. Bodies unchanged.
Production code was not touched.

### Task 3 — `integration` CI job

Replaced the "intentionally not run" comment block in `.github/workflows/ci.yml`
with a real `integration` job: `needs: test`, writes a `.env` with
`KLAKE_STORAGE__ACCESS_KEY_ID` / `KLAKE_STORAGE__SECRET_ACCESS_KEY` (both >= 8
chars, satisfying WR-09), brings up `postgres`, `minio`, `qdrant` via the repo's
own `docker compose` (single source of truth for services — MinIO's
`server /data` command can't be expressed via Actions `services:` blocks),
polls `docker compose ps <services> --format '{{.Name}} {{.Health}}'` for
health (no blind `sleep`), then runs
`uv run pytest tests/integration -m "not browser" -q` — same browser exclusion
the unit job already uses (no Playwright browsers in CI). Dumps
`docker compose logs` on failure.

I could not execute the GitHub Actions workflow (no CI runner access). I
validated it by parsing the YAML with `yaml.safe_load` and checking every
`needs:` reference resolves to a real job id (`integration needs [test]`).
I did not run this job end-to-end in an Actions runner, so I am not claiming
it has passed in CI — only that the job graph and YAML are well-formed and it
mirrors the exact local path (`make up && make test-integration`) that I did
verify against the running compose stack.

### Task 4 — `aviation` reference pack

Replaced the `# TODO` / `_ = text  # placeholder` stub in
`domains/aviation/validators/validate.py` with three real, stdlib-only checks:
1. Empty document text → fatal error.
2. No aviation term/identifier detected (N-number regex, `14 CFR 91.3`-style
   citation regex, or domain vocabulary from `taxonomy.yaml`'s entity types
   plus common operational terms) → warning `no_aviation_terms_detected`.
3. A bare `CFR` mention that doesn't match the well-formed citation shape →
   warning `malformed_regulation_citation`.

The module stays self-contained (stdlib-only, no `knowledge_lake` imports —
Pitfall 7, since it's loaded via `importlib.util` without package context).

Fixed the scaffold grammar bug: `scaffold.py`'s `qa_generation.j2` template
hardcoded `"fine-tuning a __DOMAIN__-domain language model"`, which is wrong
for every vowel-initial domain name. Added an `__ARTICLE__` sentinel and a
`_article_for(name)` helper (vowel-letter heuristic) wired into `_render()`,
and applied the same fix to the already-generated
`domains/aviation/prompts/qa_generation.j2` ("an aviation-domain language
model").

### Task 5 — Generic domain pack coverage

Added `tests/unit/test_domain_packs.py`: discovers every `domains/{name}/`
directory containing a `domain.yaml` at collection time (not hardcoded) and
parametrizes 6 assertions over each discovered pack via a `pytest.fixture(params=...)`
loader fixture: manifest name matches directory, `sources.yaml` parses into
`SourceEntry` models, both `enrich.j2` and `qa_generation.j2` render, and the
validator loads with a callable `validate_document()`. A sanity test asserts
at least 2 packs are discovered (and specifically `healthcare` + `aviation`)
so a discovery-logic bug can't silently reduce this to a no-op suite.
Healthcare-specific tests (`test_healthcare_*.py`, `test_domain_loader.py`)
were left untouched, as scoped.
