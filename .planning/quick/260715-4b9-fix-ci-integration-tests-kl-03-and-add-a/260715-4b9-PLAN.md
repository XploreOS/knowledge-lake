# Quick Task 260715-4b9: Fix CI integration tests (KL-03) + aviation reference pack

## Goal

Close **KL-03** from `.planning/E2E-GAP-ANALYSIS.md`: get `tests/integration`
(211 tests) actually running and green, both locally and in CI. Then commit the
`aviation` E2E fixture as a second reference domain pack — with coverage, so it
does not rot the way the integration suite did.

## Findings from investigation (already established — do not re-derive)

**1. `klake_test` database is never created by anything.**
`infra/postgres/init.sql` creates `dagster_storage` and `litellm_storage` but not
`klake_test`. Tests expect it — `test_migrations.py:4` calls it "the compose
klake_test DB". So the documented local path (`make up && make test-integration`)
is broken too, not just CI. This causes all 13 errors:
`psycopg.OperationalError: database "klake_test" does not exist`.

**2. Six integration mocks have a stale `put_raw` signature.**
Production is:
```python
def put_raw(self, source_id, data, ext, session, mime_type=None, domain=None, tags=None)
```
The mocks are:
```python
def mock_put_raw(source_id, data, ext, session, mime_type=None)
```
→ `TypeError: mock_put_raw() got an unexpected keyword argument 'domain'`

Exact sites (2 each):
- `tests/integration/test_upload.py:71,126`
- `tests/integration/test_ingest_url_dedup.py:65,132`
- `tests/integration/test_dedup_noop.py:73,142`

Note `tests/unit/test_put_raw_domain.py` already exists — a unit test *was* added
when `domain` landed, but the integration mocks were never updated and CI never
ran them. That is the whole KL-03 story in one diff.

**3. The CI exclusion was deliberate, not an oversight.**
`.github/workflows/ci.yml` ends with a comment block stating integration tests
are "intentionally not run on every push to keep CI fast and reliable... enable
the job below by adding the required `services:` and environment configuration."
This task carries out that documented intent. Replace the comment block with the
real job — do not leave it contradicting the new job.

**4. Compose is the single source of truth for services.**
Do NOT hand-roll GitHub Actions `services:` blocks. The MinIO image needs a
`server /data` command, which Actions `services:` cannot override. Use the repo's
own `docker compose` in CI — the same path `make up && make test-integration`
uses locally. One source of truth, and it exercises `init.sql` (which Task 1
fixes).
