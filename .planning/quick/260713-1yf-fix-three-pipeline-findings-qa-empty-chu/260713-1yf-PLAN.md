<objective>
Fix three independent findings from the functional-medicine end-to-end test, each as a self-contained atomic change with its own regression test and commit.

1. QA-generation reads empty chunk text → persist chunk text to object storage at chunk time and read it back via storage_uri in QA generation.
2. `discover`/`crawl` do not inherit the active domain → propagate a domain (explicit `--domain` or the active `settings.domain.domain_name`) onto registered sources.
3. Anti-bot/CAPTCHA challenge pages pass the quality gate → add a deterministic challenge-phrase filter to the parse quality gate that rejects them before indexing.

Purpose: Restore dataset grounding, correct domain partitioning of discovered/crawled data, and stop challenge pages from poisoning the index.
Output: Modified pipeline/CLI/quality modules + new challenge module + unit regression tests. No Alembic migration (existing `storage_uri` / `Source.config` columns are reused).
</objective>
