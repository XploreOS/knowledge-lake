<objective>
Fix the `export_rag_corpus` bug that emits empty `text` for every row of the gold RAG-corpus Parquet.

Root cause (identical to the just-fixed Finding-1 QA bug): chunk text lives in object storage at `chunk.storage_uri`, not in `chunk.metadata_`. The current code reads `meta.get("text", "")`, but a chunk artifact's `metadata_` only holds `{is_table, oversized, heading_prefix}` — no `text` key — so every row's text is empty (confirmed live: 69/69 rows empty). The QA path was fixed to read text back from storage; this rag-corpus consumer was missed.

Purpose: Restore grounded chunk text in the exported RAG corpus so the gold Parquet is usable for retrieval/eval, while preserving the existing information-disclosure allow-list guard and leaving the other export functions and the contamination hard gate untouched.
Output: Modified `export_rag_corpus` in `src/knowledge_lake/pipeline/export.py` + regression tests in `tests/unit/test_export.py`. No new dependency, no schema change (reuses the existing `chunk.storage_uri` column and the existing `uri_to_key` helper).
</objective>
