<objective>
Add Scrapy as an alternative, config-swappable crawler plugin for structured/enumerable sites, running each crawl in a **child process** to dodge Twisted's `ReactorNotRestartable` limitation, and fill in the auto-selection probe so a sitemap-bearing site routes to Scrapy automatically. The adapter maps Scrapy's external-process lifecycle onto the existing `start_crawl/poll_status/get_results` contract, writing one JSON object per page to a JSONL file the parent parses.

Purpose: INGEST-05 (crawl structured sites with Scrapy as an alternative crawler plugin) + the sitemap branch of D-04 auto-selection.
Output: ScrapyAdapter, scrapy_spider child module, auto-selection probe in crawl/select.py, `--crawler scrapy` wiring.
</objective>

## Phase Goal (slice framing)

**As a** knowledge-lake operator, **I want to** point the same `klake crawl` at a structured, sitemap-driven site and have it use Scrapy — automatically or by config — **so that** high-volume enumerable sites are crawled with Scrapy's mature middleware while every page still lands as raw+bronze artifacts, and repeated crawls in one running service never crash the reactor.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| adapter → child process | JSONL output from the child is parsed by the parent (trusted-code boundary, but bound size) |
| Scrapy spider → followed link | Every followed URL is attacker-influenceable |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-14 | DoS / crash (ReactorNotRestartable) | ScrapyAdapter | high | mitigate | Run each Scrapy crawl in a fresh subprocess (Pitfall 1); never call CrawlerProcess.start() in the parent. |
| T-02-15 | Information Disclosure (SSRF via followed link) | scrapy_spider | high | mitigate | validate_public_url(source_url) before spawn; ROBOTSTXT_OBEY + same-domain link scoping; re-validate followed URLs. |
| T-02-16 | Repudiation / legal (robots bypass) | scrapy middleware | high | mitigate | ROBOTSTXT_OBEY=True; robots-skipped pages recorded as robots_blocked in results. |
| T-02-17 | DoS (oversized response / self-inflicted) | Scrapy settings | medium | mitigate | DOWNLOAD_MAXSIZE 50 MB + AUTOTHROTTLE + resolved per-host delay from config. |
| T-02-SC | Tampering (supply chain) | scrapy==2.16.0 | low | accept | Approved in RESEARCH Package Legitimacy Audit (10+ yrs, very high downloads). No [ASSUMED]/[SUS]/[SLOP]. |
</threat_model>
