<objective>
Implement `dedup_chunks()` — the router that annotates every chunk with its `text_sha256`/`point_id`, atomically claims each one against the corpus-wide ledger (via Plan 21-01's `claim_dedup_ledger_entry`), and partitions the input into `new` (first-seen text, embed it) and `duplicates` (already-seen text, no embedding needed) — extending `pipeline/dedup.py` from Plan 21-02's pure-function baseline.

Purpose: This is DEDUP-01's core mechanism — the "dedup stage between chunk and embed" the roadmap specifies. It is deliberately its own function (not folded into `chunk()` or `embed()`, D-16) so it runs strictly after Phase 20's substance gate (which lives inside `chunk()`) and strictly before `embed()`, preserving the "L3 before L4" ordering the milestone's IDF-inversion rationale depends on.

Output: `dedup_chunks(chunks, parsed_artifact_id, source_id, *, collection, settings=None) -> dict`, added to `pipeline/dedup.py`, proven end-to-end (real ledger claims via the SQLite unit-test harness, not just pure-function assertions) in `tests/unit/test_index_dedup.py`.
</objective>
