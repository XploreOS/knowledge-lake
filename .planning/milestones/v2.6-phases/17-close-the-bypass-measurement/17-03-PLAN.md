<objective>
Close the CLI half of the clean-stage bypass: insert `clean()` between `parse()` and `chunk()` inside `pipeline/process.py::process_crawled` — the single function shared by CLI, API, and MCP entry points (D-03 "one function, many callers"). Purpose: CLEAN-02's acceptance criterion requires `klake process` to produce chunks from cleaned text, with parity to the Dagster path. Since `process_crawled` already holds `parsed_doc` in memory from its own `parse()` call, threading it into `clean(parsed_doc=...)` costs zero extra I/O — matching RESEARCH.md's `process_crawled` parity code example exactly.
Output: `process_crawled` calls `clean()` and passes the cleaned doc to `chunk()`; a new test file proves the call order and error-handling parity.
