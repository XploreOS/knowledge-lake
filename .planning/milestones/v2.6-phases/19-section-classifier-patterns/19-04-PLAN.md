<objective>
Wire `classify_sections()` (CLEAN-04) into `clean.py`'s existing `_clean_sections()`/`clean()` flow: a
dedicated, pure annotation function that computes substance signals and an `is_boilerplate` decision per
section (using Plan 19-01's predicate module and Plan 19-02's `DomainFilters` allowlist), then actually
DROPS sections classified as boilerplate from the returned `cleaned_doc.sections` ‚Äî the behavioral change
CLEAN-04 requires, superseding Phase 17's placeholder "annotate all, drop none" behavior.

Purpose: This is the integration point where Plans 19-01/02/03's mechanisms become load-bearing: a
document's nav/footer/junk sections are actually removed from what `chunk_document`/`tree_index_document`/
`enrich_document` receive, while clinical content protected by the domain allowlist is never dropped.

Output: `classify_sections()` in `clean.py`; `_clean_sections()`/`clean()` updated to consume it and
persist `section_annotations`; 4 existing Phase-17 tests updated to reflect the new drop-on-classify
behavior; new tests proving nav/footer removal, allowlist override, and section_annotations persistence.
</objective>
