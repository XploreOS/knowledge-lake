    unchanged), add `BOILERPLATE_PATTERNS.extend([...])` with exactly 5 new `re.compile(...)` entries, in
    this order, using this exact regex source (case-insensitive `(?i)`, multiline `(?m)` where the pattern
    is line-anchored with `^`/`$`):
    (1) Navigation: `r"(?im)^(?:main menu|breadcrumbs?|skip to footer|jump to navigation|back to top|search this site|toggle navigation)\s*$"`
    (full-line-only, extends the existing single-word nav pattern's coverage).
    (2) Terms-of-service: `r"(?im)^.*(?:terms of service|terms and conditions|terms of use)\b.*$"`
    (matches and removes the WHOLE line containing the phrase anywhere in it, not just the phrase).
    (3) Marketing/enrollment CTAs: `r"(?im)^(?:enroll now|sign up today|register for .*|subscribe now|get started for free|schedule a demo|contact sales)[^\n]*$"`
    (full-line-anchored).
    (4) Cookie consent (additional phrasing beyond the existing pattern): `r"(?im)^.*(?:we use cookies|manage cookie preferences|cookie settings)\b.*$"`.
    (5) Government disclaimer: `r"(?im)^(?:this website is not a substitute for professional medical advice|for official use only|privacy policy|accessibility statement|no fear act|foia)[^\n]*$"`
    (deliberately anchored to these SPECIFIC multi-word phrases, not a generic "disclaimer"/"warning"
    keyword — this narrowness is what satisfies the must_haves.prohibitions entry above about not
    stripping genuine clinical safety text). Use `.extend([...])` as a single call with a list literal
    (not five separate `.append()` calls) — either is additive-only, but one `.extend()` call is
    equivalent and cleaner. Update the module's header comment (lines 42-46) to note the list now has 9
    entries covering all 5 CLEAN-05 audit categories plus the original 4.
  </action>
