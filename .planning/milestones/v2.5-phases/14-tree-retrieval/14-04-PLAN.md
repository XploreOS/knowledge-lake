<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI/query → tree_search | The query string is user input flowing into stage-1 search and (in llm mode) the retriever prompt |
| S3 tree JSON → _dict_to_tree | Loaded tree bytes are deserialized into the typed contract; malformed/oversized JSON is a parsing surface |
| stage-2 concurrency → S3 | Parallel loads are bounded so a large shortlist cannot exhaust connections/memory |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-14-09 | Denial of Service | parallel S3 tree loading | high | mitigate | asyncio.Semaphore(settings.tree_search.concurrency) bounds in-flight loads; max_docs (default 3) caps the shortlist; a single non-nested asyncio.run drives the batch (D-10, CR-02). Verified by test_parallel_load_and_skip. |
| T-14-10 | Tampering | _dict_to_tree deserialization of S3 JSON | medium | mitigate | Parse with orjson.loads and reconstruct via the typed TreeIndex/TreeNode contract (fields read explicitly, not **spread); a missing/failed load returns None and is skipped (D-09) rather than crashing the query. |
| T-14-11 | Denial of Service | missing tree_index artifact | low | mitigate | get_child_artifact_by_type returning None is logged and skipped — one document's missing tree never fails the whole query (D-09). |
| T-14-12 | Information Disclosure | CLI hit rendering | low | accept | The CLI renders only fields already present in Hit.payload (document, section_path, pages, score); no DB lookup or secret exposure. Accept at ASVS L1. |
| T-14-SC | Tampering | npm/pip/cargo installs | low | accept | No new packages added; orchestrator + CLI reuse existing dependencies (asyncio, orjson, typer all already present). Package Legitimacy Audit is a no-op per RESEARCH.md. |
</threat_model>
