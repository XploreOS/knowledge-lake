<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| tree node text → LLM prompt | Node title/summary originate from ingested documents (untrusted); they are interpolated into the LLM-nav prompt (prompt-injection surface) |
| LLM response → Hit selection | The model's returned node_ids are untrusted and must be validated before use |
| settings.retriever → entry-point load | The resolved name selects which class to import and execute |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-14-05 | Tampering | LLM-nav prompt (node text interpolation) | high | mitigate | Injection-resistant system prompt (mirror _SUMMARY_SYSTEM_PROMPT) instructing the model to treat node text as untrusted data; bound each node excerpt to ~512 chars (mirror _NODE_EXCERPT_CHARS); validate the response with a bounded Pydantic NavResult (node_ids list with Field max_length); discard node_ids not present in the tree. |
| T-14-06 | Denial of Service | LLM-nav cost / runaway spend | high | mitigate | Budget gate reads get_llm_spend(scope="tree_search") and returns the heuristic result when spend >= budget_usd; record_llm_spend after each call; temperature=0.0; single completion per search. Isolated scope prevents cross-contaminating Phase-13 tree_index or global budgets (D-07). |
| T-14-07 | Elevation of Privilege | resolver.get_retriever entry-point name | medium | mitigate | settings.retriever is regex-validated upstream (T-14-03, Plan 14-02); _resolve_with_kwargs raises LookupError on unknown names rather than importing arbitrary modules. |
| T-14-08 | Information Disclosure | litellm gateway credentials | low | accept | litellm_url/api_key are constructor-injected (CR-03, no os.environ in builtin); defaults are local-noauth; production values flow from Settings. Accept at ASVS L1. |
| T-14-SC | Tampering | npm/pip/cargo installs | low | accept | No new packages added; the editable reinstall (uv pip install -e .) only refreshes this repo's own dist-info to expose the new entry-point group. litellm is already a project dependency. Package Legitimacy Audit is a no-op per RESEARCH.md (pageindex is NOT imported). |
</threat_model>
