# Phase 16 — UI Design Contract

> This phase is **backend-only**. There are no browser-rendered visual components,
> no CSS, no JavaScript frontend, and no interactive UI. The phase produces a
> Python pipeline module (`pipeline/wiki.py`), a Typer CLI command (`klake export-wiki`),
> and a FastAPI REST endpoint (`POST /export-wiki`). All output is Markdown files
> stored in S3 for consumption by external wiki tools (Obsidian, Notion).
>
> This contract documents the **interface contracts** (API response schema, CLI
> output format, generated Markdown structure) rather than visual design tokens.
> All 6 visual design dimensions are marked N/A.

## API Response Contract

### POST /export-wiki

**Request body (JSON):**

```json
{
  "domain": "string (required)",
  "force": "boolean (default: false)"
}
```

**Success response (200 OK):**

```json
{
  "domain": "healthcare",
  "pages_created": 12,
  "pages_updated": 3,
  "pages_unchanged": 15,
  "pages_total": 30,
  "concept_pages": 8,
  "document_pages": 21,
  "index_page": true,
  "wiki_prefix": "gold/healthcare/wiki/",
  "manifest_key": "gold/healthcare/wiki/_manifest.json",
  "incremental": true
}
```

**Error responses:**

| Status | Condition | Body |
|--------|-----------|------|
| 422 | Missing `domain` parameter | `{"detail": [{"loc": ["body", "domain"], "msg": "Field required", "type": "missing"}]}` |
| 404 | Domain has no ingested documents | `{"detail": "No enriched documents found for domain 'xyz'. Ingest and enrich documents first."}` |
| 500 | Storage write failure | `{"detail": "Wiki export failed: {error_message}"}` |

## CLI Output Contract

### klake export-wiki

**Normal output (stdout):**

```
Wiki export: healthcare
  Documents: 21 pages (12 new, 3 updated, 6 unchanged)
  Concepts:  8 pages (5 new, 2 updated, 1 unchanged)
  Index:     1 page (updated)
  Total:     30 pages written to gold/healthcare/wiki/
  Manifest:  gold/healthcare/wiki/_manifest.json

Done in 2.3s
```

**Dry-run output (--dry-run flag):**

```
Wiki export: healthcare (dry run)
  Would create: 12 document pages, 5 concept pages
  Would update: 3 document pages, 2 concept pages
  Would skip:   6 document pages, 1 concept page
  Index page:   would update

No files written (dry run mode).
```

**Error output (stderr):**

```
Error: No enriched documents found for domain 'xyz'.
Hint: Run 'klake enrich' on ingested documents first.
```

**Exit codes:**

| Code | Meaning |
|------|---------|
| 0 | Success |
| 1 | Runtime error (storage failure, DB connection) |
| 2 | Validation error (missing domain, no documents) |

## Copywriting Contract

| Element | Copy | Surface |
|---------|------|---------|
| CLI success message | `Done in {N}s` | Terminal stdout |
| CLI no-documents error | `No enriched documents found for domain '{domain}'.` | Terminal stderr |
| CLI hint after error | `Run 'klake enrich' on ingested documents first.` | Terminal stderr |
| API no-documents error | `No enriched documents found for domain '{domain}'. Ingest and enrich documents first.` | JSON response body |
| Dry-run footer | `No files written (dry run mode).` | Terminal stdout |
| Wiki page footer | `Generated by Knowledge Lake wiki export` | Markdown footer |
| Force rebuild note | `Full rebuild forced (--force flag)` | Terminal stdout |

**No destructive actions in this phase.** Wiki export is additive/idempotent
(writes new pages, updates changed pages, never deletes). The `--force` flag
triggers a full rebuild but does not destroy data — it overwrites with
freshly computed content.
