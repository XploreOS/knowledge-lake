<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Registry DB -> wiki.py | Enrichment metadata from DB could contain unexpected characters in entity/title strings |
| wiki.py -> S3 keys | Page slugs derived from user-influenced titles form S3 object keys |
| S3 manifest -> wiki.py | Manifest JSON read from S3 is untrusted input (could be tampered) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-16-01 | Tampering | S3 key construction via entity/title strings | medium | mitigate | slugify() strips all non-alphanumeric chars to hyphens; keys constructed only from _GOLD_PREFIX + domain_seg + slugified strings; no raw user input in keys |
| T-16-02 | Tampering | Domain name in S3 key path | medium | mitigate | Domain name validated by existing _DOMAIN_NAME_RE regex in api/app.py before reaching compile_wiki; CLI validates via typer option |
| T-16-03 | Information Disclosure | Markdown page content from enrichment metadata | low | accept | Content sourced from EnrichmentResult (bounded by Pydantic max_length validators T-04-04/T-04-06); no raw document text beyond excerpt_chars |
| T-16-04 | Denial of Service | Unbounded corpus size | low | mitigate | IDF threshold limits concept page explosion; structured log warning for large corpora (>1000 docs); operator-triggered batch (not user-facing) |
| T-16-05 | Tampering | Manifest JSON deserialization | low | mitigate | Manifest loaded via orjson.loads (safe JSON parser); schema validated before use; malformed manifest triggers full rebuild with warning |
</threat_model>
