  <action>
Three files modified. Each gains `route` and switches from calling search() directly to calling routed_search().

**api/app.py — search_endpoint (per D-03, Pitfall 1):**
Add a `route` Query parameter directly after the existing `mode` Query param (~L220). Use: `route: str | None = Query(default=None, pattern=r"^(chunk|tree|two_stage|auto)$", description="Retrieval route: chunk|tree|two_stage|auto. Defaults to KLAKE_ROUTER__DEFAULT_ROUTE (auto). Unrecognised value rejected with 422 (ASVS V5).")`. The pattern= attribute makes FastAPI auto-reject invalid values with 422 before the handler body runs (fail-closed, T-15-01).

Change the import: replace `from knowledge_lake.pipeline.search import search` with `from knowledge_lake.pipeline.route import routed_search` (or add it alongside if search is used elsewhere in the file — grep first). At the call site where `search(query=q, ...)` is invoked, switch to `routed_search(query=q, route=route, ...)` forwarding all existing filter kwargs unchanged. The `mode` kwarg continues to be forwarded (it controls hybrid/dense/sparse within whichever path is selected).

If there is a structlog/logger event that logs the search call parameters, add `route=route` to it.

**cli/app.py — cmd_search (per D-03, mirrors --mode pattern):**
Add a `--route` Option in the cmd_search function signature after the existing `mode` Option (~L661): `route: str | None = typer.Option(None, "--route", help="Retrieval route: chunk|tree|two_stage|auto (default from KLAKE_ROUTER__DEFAULT_ROUTE, else auto).")`.

Add a VALID_ROUTES guard immediately after the existing VALID_MODES guard (~L688): define `VALID_ROUTES = {"chunk", "tree", "two_stage", "auto"}`, check `if route is not None and route not in VALID_ROUTES:`, print error to stderr via `typer.echo(..., err=True)`, raise `typer.Exit(code=1)`.

Change the import from `from knowledge_lake.pipeline.search import search` to `from knowledge_lake.pipeline.route import routed_search`. At the call site (~L692), switch `search(query, ...)` to `routed_search(query, route=route, ...)` forwarding all existing kwargs (collection, top_k, domain, document_type, min_quality_score, source_name, format, source_id, tags=tag, mode=mode).

**agent/registry.py — _search_handler (per D-03, D-08, Pitfall 1):**
Add `route: str | None = None` to the _search_handler function signature (after `mode`, ~L151). Change the import from `from knowledge_lake.pipeline.search import search` to `from knowledge_lake.pipeline.route import routed_search`. At the call site, switch `search(query=q, ...)` to `routed_search(query=q, route=route, ...)` forwarding all existing kwargs. The MCP server does model.model_dump(exclude_none=True) then calls fn(**kwargs) — without route in the signature, a TypeError would be raised when a client passes route (Pitfall 1).

Optionally extend the search ToolDef description (~L243-247) to mention route selection alongside mode. Not required by ROUTE-04 but improves agent tool-use.
