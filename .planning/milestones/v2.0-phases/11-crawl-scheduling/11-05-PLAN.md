<objective>
Wire the first-ever Dagster `@sensor` for Phase 11: `recrawl_sensor` evaluates each scheduled source's cron due-ness and emits deterministic, deduplicated `RunRequest`s targeting a new op-based `recrawl_source_job`, whose single op runs the change-gated `recrawl_source()` (Plan 11-03). Per-source concurrency=1 is enforced by switching the run coordinator. The sensor stays side-effect-free apart from its cursor watermark; all DB writes happen in the op.

Purpose: The lake re-crawls sources on schedule without duplicate runs, tick storms, or overlapping same-source crawls.
Output: `dagster_defs/sensors.py` (sensor + op + job + config), Definitions registration, and the `QueuedRunCoordinator` swap.

Decisions implemented: D-04 (cron due-check base = last_crawled_at or created_at), D-12 (@sensor in sensors.py registered via Definitions, minimum_interval_seconds=60), D-13 (op-based recrawl_source_job reusing recrawl_source, no crawl logic duplicated), D-14 (deterministic run_key from the cron fire timestamp), D-15 (cursor watermark via update_cursor), D-16 (per-source concurrency=1 via QueuedRunCoordinator + klake/source tag), D-17 (DB writes only in the op, sensor side-effect-free).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Sensor → Dagster run storage | Uncontrolled RunRequest emission could flood the run queue (tick storm). |
| Sensor → registry (read-only) | The sensor reads scheduled sources; it must not write. |
| Run coordinator → executor | Per-source serialization prevents a slow crawl overlapping its own next tick. |

## STRIDE Threat Register (phase-level; this plan implements the tick-storm + sensor-integrity mitigations)

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-11-TICKSTORM | Denial of Service | recrawl_sensor + run coordinator | high | mitigate | Deterministic run_key = f"{sid}:{fire.isoformat()}" (Dagster dedups) + minimum_interval_seconds=60 + QueuedRunCoordinator per-source limit=1 via klake/source tag. Asserted by test_recrawl_sensor. |
| T-11-WRITE | Tampering / Integrity | recrawl_sensor | medium | mitigate | Sensor side-effect-free except context.update_cursor; all DB writes happen in the op via recrawl_source/touch_source_crawl. |
| T-11-CRON | Denial of Service (V5 input) | cron due-check | medium | mitigate | Cron parsed by Dagster's vendored engine (get_next_cron_tick); only pre-validated schedules reach the sensor (Plan 11-04). |
| T-11-SSRF | Information Disclosure | recrawl_op → recrawl_source | high | mitigate | The op calls recrawl_source, which runs validate_public_url before any HTTP (Plan 11-03). |
| T-11-SC | Tampering (supply chain) | dependency set | high | mitigate | No new package added; cron helpers come from Dagster's vendored `dagster._utils.schedules`, never the SUS-flagged standalone croniter. |
</threat_model>
