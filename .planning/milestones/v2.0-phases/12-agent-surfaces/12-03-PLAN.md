<objective>
Make the Pydantic layer the single schema source of truth for every tool. Extend `SearchParams` to cover all `search()` filters, and add the small input models for the tools that have no existing request schema — reusing existing schemas everywhere one already fits (D-02).

Purpose: SKILL-03 "no drift" only works if each tool's shape is defined once as a Pydantic model. `SearchParams` currently under-covers `search()` (Pitfall 4), and `stats`/`process_crawled`/`list_sources`/`lineage`/`ingest_url`/`crawl_all` have no request schema.

Output: an extended `SearchParams` plus new `StatsInput`, `ProcessCrawledInput`, `ListSourcesInput`, `LineageInput`, `IngestUrlInput`, `CrawlAllInput` in `api/schemas.py`, each field-aligned to its service-function kwargs.
</objective>
