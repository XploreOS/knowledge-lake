<objective>
FIRST-TASK GATE (ROADMAP research note, D-07/D-08): land the fd-level stdout-lockdown shim and its self-test in stdio mode ONLY, before any MCP tool logic exists.

Purpose: structlog's `PrintLoggerFactory()` (`__init__.py:42`, no `file=`) writes to stdout — the exact channel the MCP stdio JSON-RPC framing owns. Any stray log/print/C-extension byte corrupts the stream and kills the session. An fd-level `dup2` redirect (not merely reconfiguring the Python logger) is the robust fix; the self-test proves it deterministically.

Output: `src/knowledge_lake/agent/__init__.py`, `src/knowledge_lake/agent/stdio.py` with `run_stdio(server, init_opts)`, and a passing `tests/integration/test_stdio_lockdown.py` that spawns a stdio server with a trivial tool and asserts only JSON-RPC reached stdout.
</objective>

Implement an async `run_stdio(server, init_opts)` that, in this exact order: (1) preserve the real JSON-RPC channel with `os.dup(1)` BEFORE any redirect; (2) redirect process fd 1 to stderr with `os.dup2(2, 1)` and set `sys.stdout = sys.stderr` so Python-level print goes to stderr too; (3) reconfigure logging to stderr — re-run `structlog.configure(...)` with `logger_factory=structlog.PrintLoggerFactory(file=sys.stderr)` (mirroring the processor list at `__init__.py:42`), `logging.basicConfig(stream=sys.stderr)`, and `logging.captureWarnings(True)`; (4) build a clean binary stream on the PRESERVED fd (`TextIOWrapper(os.fdopen(real_fd, "wb", buffering=0))`, wrapped via `anyio.wrap_file`) and pass it explicitly as `stdio_server(stdout=preserved)`, then `await server.run(read, write, init_opts)`. The `stdout=` argument is mandatory — if the SDK grabs `sys.stdout.buffer` after the dup2, JSON-RPC would be written to stderr. This lockdown is stdio-only; HTTP mode never calls it (D-08).
  </action>
