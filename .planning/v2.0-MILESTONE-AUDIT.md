## Cross-Phase Integration (WIRED — 0 broken)

- **MCP tools → pipeline functions (Phase 12 → 7–11):** all 11 registry handlers dispatch
  `input_model(**arguments) → fn(**model.model_dump(exclude_none=True))` to real, correctly-signed
  `pipeline/*.py` / `lineage` functions — never proxying REST (MCP-01 holds). The one historical
  mismatch (`add_source → register_source`) is confirmed fixed (commit `2d73db1`).
- **Search filter path (PAYLOAD-02 → RETR-03 → MCP):** `source_name/format/tags/source_id` + `mode`
  flow identically through CLI, REST, and MCP into one `pipeline.search.search()`; default mode
  `hybrid`, fail-closed on unknown mode.
- **Scheduling (SCHED → crawl):** `recrawl_sensor → recrawl_source` compares normalized silver-stage
  text with `max_staleness_days`, deterministic `run_key` + cursor watermark. Wired (runtime note:
  a running Dagster daemon needs a code-location reload to pick up the sensor).
- **Single schema source of truth (SKILL-03):** OpenAPI, OpenAI tool defs, and MCP `inputSchema` all
  derive from the same `TOOLS[*].input_model` — `stdio == http == openapi == openai` by construction.
