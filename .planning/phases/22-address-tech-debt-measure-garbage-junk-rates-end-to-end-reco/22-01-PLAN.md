<objective>
Extend `pipeline/quality_audit.py` with a new `run_full_pipeline_audit()` function that produces the milestone's two originally-audited, literal-unit measurements — chunk-level garbage rate (criterion #1) and export-level junk rate (criterion #2) — reusing `clean()`, `chunk()`, and `export_rag_corpus()` unmodified, with zero new gate logic (D-01). Fixes a real pre-existing bug (Pitfall 1): `run_quality_audit()`'s `clean()` call never threads `domain_filters`, so a clinical-code section could be stripped before any downstream gate ever protects it. Solves the phase's central design constraint (D-04): a naive domain-wide re-scan of `export_rag_corpus()`'s chunk population would dilute the measurement with ~4,512 pre-v2.6 chunks that default `substance_passed=True` — this plan scopes the export-junk measurement to only the chunk IDs this run itself produces. Also documents the export dedup-awareness finding (D-07) as an accepted, non-code design boundary.

Purpose: Close tech-debt item #1 from `.planning/v2.6-MILESTONE-AUDIT.md` — the milestone's two quantitative success criteria have never been measured in their literal units against real data.
Output: `run_full_pipeline_audit()` (new), a corrected `run_quality_audit()`, an updated `export.py` docstring, and five new/extended unit tests proving the chunk-tally and D-04-scoping behavior against seeded fixture data.
