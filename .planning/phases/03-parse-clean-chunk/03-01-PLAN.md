**src/knowledge_lake/quality/scorer.py — new module implementing D-04 quality scoring:**

Module docstring: "Heuristic quality scorer for ParsedDoc output (PARSE-04, D-04). Deterministic-first: always computes a weighted heuristic score. Optional LLM spot-check fires only in the configurable gray zone."

Define `compute_quality_score(parsed_doc: ParsedDoc, mime_type: str = "", settings=None) -> float`:
- Weighted heuristics with weights summing to 1.0:
  - text_length_score = min(len(parsed_doc.text) / 200, 1.0) — weight 0.35; at least 200 chars expected
  - section_score = min(len(parsed_doc.sections) / 3, 1.0) — weight 0.20; at least 3 sections is good
  - encoding_score = 1.0 - min(parsed_doc.text.count("�") / max(len(parsed_doc.text), 1) * 100, 1.0) — weight 0.25; unicode replacement char ratio
  - empty_section_score: if sections exist, 1.0 - (empty_count / len(sections)); else 0.5 — weight 0.20
- Return weighted sum as float between 0.0 and 1.0
- Log score with structlog at debug level including text_len, section_count, mime_type

Define `maybe_llm_spot_check(parsed_doc: ParsedDoc, score: float, settings) -> float`:
- Takes the heuristic score, gray zone from settings.parse.quality_gray_zone, llm_spot_check flag from settings.parse.llm_spot_check
- If llm_spot_check is False OR score is outside the gray zone: return score unchanged
- If inside gray zone: make a single LiteLLM call via `settings.litellm_url` with the `cheap_model` task alias asking "Rate the coherence of this text 0.0 to 1.0 as JSON {score: float}"; parse the response; return the LLM score
- Wrap the LiteLLM call in a try/except; on any error log a warning and return the original heuristic score unchanged
- Use `import litellm` inside the function body (lazy import) so tests that do not exercise the LLM path do not require a running LiteLLM proxy

Add function `parse_with_fallback(raw: bytes, mime_type: str, *, settings: "Settings") -> tuple[ParsedDoc, str, float]`:

Docstring: "Try parsers in settings.parse.chain order. Stop on first success (D-02). Continue on exception or quality gate failure (D-01). Returns (ParsedDoc, parser_name_used, quality_score)."

Loop over `settings.parse.chain`:
1. Attempt `resolve(GROUP_PARSERS, parser_name)` — on `LookupError` log warning "parser_not_available" and `continue`
2. Check `parser.can_parse(mime_type)` — if False, `continue` (parser skips unsupported types silently)
3. Try `parser.parse(raw, mime_type)` — on any `Exception` log warning "parser_failed" with exc info, `continue`
4. Call `compute_quality_score(parsed_doc, mime_type)`
5. Call `maybe_llm_spot_check(parsed_doc, score, settings)` to potentially refine with LLM
6. If final score >= `settings.parse.quality_threshold`: return `(parsed_doc, parser_name, final_score)`
7. Else log warning "parser_low_quality" with score and continue

After loop exhausted: raise `ValueError(f"All parsers in chain exhausted for mime_type={mime_type!r}. Chain: {settings.parse.chain}")`

Keep existing `get_parser()` function unchanged (used by existing tests and the Phase 1 pipeline path).
