**src/knowledge_lake/pipeline/chunk.py — replace internals, preserve registration shell:**

Remove `MAX_CHUNK_CHARS = 1200` constant.

Add module-level tiktoken encoder initialization:
`import tiktoken as _tiktoken`
`_encoder = _tiktoken.get_encoding("cl100k_base")`

This is the module-level cache (Pitfall 2 avoidance — never re-instantiate per call).

Add `token_count(text: str) -> int` function: `return len(_encoder.encode(text))`

Add `_split_sentences(text: str) -> list[str]` function:
Use `re.split(r'(?<=[.!?])\s+(?=[A-Z])', text)` as the primary split. This handles common sentence boundaries while guarding against abbreviations (the positive lookbehind requires punctuation). Filter out empty strings. Return the list. If the list is empty, return `[text]`.

Add `chunk_section(text: str, max_tokens: int, overlap_tokens: int, heading_prefix: str = "") -> list[str]` function:
- If `token_count(text) <= max_tokens`: return `[text]` (single chunk, no prefix in text — prefix stored in metadata)
- Split text into sentences using `_split_sentences()`
- Accumulate sentences into chunks by calling `token_count(sent)` per sentence once (pre-compute costs list: `costs = [token_count(s) for s in sentences]`)
- Build chunks using the sliding-window approach: maintain `current_sentences: list[str]`, `current_tokens: int`
- When `current_tokens + next_cost > max_tokens` and current is non-empty: emit `" ".join(current_sentences)` as a chunk, then compute overlap: keep trailing sentences from current whose total token cost <= overlap_tokens
- Continue until all sentences are consumed
- Emit remaining as final chunk
- If no chunks were produced (single sentence > max_tokens): return `[text]` (oversized single-sentence is atomic like a table)
- Return list of chunk strings

Replace `_build_chunks(parsed_doc: ParsedDoc) -> list[dict]` entirely:

New `_build_token_chunks(parsed_doc: ParsedDoc, max_tokens: int, overlap_tokens: int, breadcrumb_depth: int) -> list[dict]`:

If no sections: emit full text as single chunk with section_path="§1", page=1, is_table=False.

For each section in parsed_doc.sections:
- Compute `heading_prefix = section.heading` if breadcrumb_depth >= 1, else `""`. For breadcrumb_depth >= 2, include parent heading if available (section_path can be used to infer nesting — for MVP, use only the immediate section heading as prefix, stored in metadata not in text)
- If `section.is_table`:
  - Emit as single atomic chunk regardless of size
  - Set `oversized = token_count(full_text) > max_tokens`
  - Chunk dict: `{"text": full_text, "section_path": section.section_path, "page": section.page, "is_table": True, "oversized": oversized, "heading_prefix": section.heading}`
- Else:
  - Combine heading + body text: `full_text = f"{section.heading}\n\n{section.text}".strip() if section.heading else section.text.strip()`
  - Skip if empty
  - Call `chunk_section(full_text, max_tokens, overlap_tokens)` to get sub-chunks
  - For sub-chunk at index i: dict `{"text": sub, "section_path": f"{section.section_path}.{i+1}" if len(subs) > 1 else section.section_path, "page": section.page, "is_table": False, "oversized": False, "heading_prefix": section.heading}`
- Append all produced dicts to results

Return results list.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| chunk text → tiktoken encoder | Arbitrary text from parsed documents enters the tokenizer |
| API /parse endpoint → pipeline | External request triggers parse stage on an already-stored artifact |
| CLI raw_artifact_id input | User-provided artifact ID looked up in registry |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-09 | Denial of Service | chunk() with oversized table section | low | accept | Oversized tables emit as single chunks with oversized=True flag; token count is only for metadata; no re-encoding loop; O(1) per table section |
| T-03-10 | Denial of Service | tiktoken.get_encoding called at module import | low | accept | Module-level caching — encoder initialized once at import time; subsequent calls are O(1) lookups; documented in chunk.py module docstring |
| T-03-11 | Injection | API /parse, /clean, /chunk artifact_id inputs | medium | mitigate | artifact_id is looked up via parameterized ORM query (get_artifact) — no raw SQL; invalid IDs return 422; IDs follow the strict prefixed UUIDv7 format validated by new_id() |
| T-03-12 | Spoofing | Dagster asset clean_document forwarding parsed_doc | low | accept | ParsedDoc is an in-memory struct produced by the trusted parse stage in the same Dagster run; not deserialized from external input |
</threat_model>
