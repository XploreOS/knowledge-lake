<objective>
Wire `dedup_chunks()` into `process_crawled()` (the CLI/API/MCP shared entry point, D-03 "one function, many callers") between the existing `chunk()` call and the existing `embed()`/`index()` calls — the first of the two call sites D-17 requires.

Purpose: Without this wiring, `dedup_chunks()`/`index()`'s new capabilities exist but are dead code on the CLI/API/MCP path — `klake process <source>` would keep embedding and indexing every chunk exactly as before, with zero dedup benefit. This closes that gap for the shared pipeline function every non-Dagster entry point uses.

Output: `process.py`'s `chunk() -> embed() -> index()` chain becomes `chunk() -> dedup_chunks() -> embed(new only) -> index(new, duplicate_chunks=duplicates)`, proven by a new test file mirroring the exact call-order/argument-identity style Phase 17's `test_process_crawled_clean.py` established for the analogous `clean()` insertion.
</objective>
