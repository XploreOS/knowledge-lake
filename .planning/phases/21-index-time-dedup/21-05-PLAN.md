<objective>
Extend `index()` with an optional `duplicate_chunks` parameter: for every deduplicated chunk, append its contributor entry to the ledger (Postgres, source of truth per D-13), mirror a capped `contributors[]` + exact `contributor_count` onto the existing Qdrant point via `set_payload()`, and self-heal (re-embed + full upsert, ledger repair) if the point has vanished out-of-band. Also switch the "new" chunks' point-ID resolution to prefer `dedup_chunks()`'s deterministic `point_id` annotation, falling back to the legacy `_strip_prefix(chunk_id)` scheme for any caller that never went through dedup.

Purpose: This is DEDUP-03's payload-preservation contract and the second half of DEDUP-02's point-ID determinism (the write side; Plan 21-02 built the pure derivation). `index()` is the single place that already owns "what lands in Qdrant" (D-19) — extending it here, rather than adding a parallel code path, is what keeps PAYLOAD-01/02 filters working for free.

Output: `index()` with a `duplicate_chunks: list[dict] | None = None` keyword parameter, the full contributor-append/cap/self-heal branch, and a `reindex_collection()` docstring update noting the accepted dual-ID-scheme transitional state (D-08).
</objective>
