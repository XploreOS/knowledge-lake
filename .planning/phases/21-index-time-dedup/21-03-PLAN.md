<objective>
Extend the `VectorStorePlugin` protocol with one new method, `set_payload(collection, point_id, payload) -> bool`, and implement it on `QdrantVectorStore` — the mechanism `index()`'s duplicate-routing branch (Plan 21-05) uses to mirror a deduplicated chunk's contributor list onto an EXISTING Qdrant point without re-embedding or overwriting its primary metadata.

Purpose: DEDUP-03 requires the single surviving point's payload to carry an additive `contributors[]` list whenever a duplicate is discovered. Qdrant's own client does not expose a "merge payload, tell me if the point existed" primitive directly — `QdrantClient.set_payload()` raises on a missing point rather than silently no-op-ing (an empirical correction RESEARCH.md made to CONTEXT.md's original assumption) — so this plan wraps that behavior into the boolean-existence contract D-24's self-healing logic needs.

Output: One new protocol method declaration and one concrete implementation, proven against a fully-mocked Qdrant client (no real server required for these tests, mirroring `tests/unit/test_qdrant_payload_indexes.py`'s established mocking convention).
</objective>
