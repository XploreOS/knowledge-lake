<objective>
Add a new `dedup_chunks` Dagster asset between `chunk_document` and `embed_chunks`, rewire `embed_chunks`/`index_chunks` to consume its output, and add it to `core_pipeline_e2e_job`'s asset selection — the second of the two call sites D-17 requires, giving the Dagster-orchestrated pipeline the same dedup behavior Plan 21-06 gives the CLI/API/MCP path.

Purpose: Without this wiring, the Dagster-orchestrated pipeline (the production orchestration path per this project's constraints — "Orchestration: Dagster from day 1") would keep embedding and indexing every chunk exactly as before, with the dedup mechanism only reachable via `klake process`. This also closes the KL-06-shaped regression risk: a new asset absent from a job's `AssetSelection` silently drops any `deps=`/data-dependency edge the job needs, even though the asset graph looks correct in the abstract.

Output: A new `dedup_chunks` asset, `embed_chunks`/`index_chunks` updated to read from it, `core_pipeline_e2e_job`'s selection tuple updated, and `test_asset_ordering.py` extended with the Pitfall-1 guard.
</objective>
