    - A Pydantic BaseModel named NodeSummaryResult with one field: summary: str with max_length=512 (mirrors EnrichmentResult). This validates the LLM output before use (ASVS V5).

    Helper function _derive_level(section_path: str) -> int: returns section_path.count(".") + 1. "§1" → 1, "§1.1" → 2, "§1.1.1" → 3. Must be deterministic and never use random/clock.

    Helper function _build_deterministic_tree(sections: list[Section], page_count: int) -> list[TreeNode]: builds the roots list using a level-stack approach:
    - Initialize stack = [] and roots = []
    - For each section in sections (in list order):
      - Derive level = _derive_level(section.section_path)
      - Derive node_id = section.section_path (stable, deterministic — Pitfall 3)
      - Create TreeNode(node_id=node_id, title=section.heading, summary=section.heading, page_start=section.page, page_end=0, level=level, section_path=section.section_path)
      - Pop stack entries with level >= current level (we are at the same or shallower depth)
      - If stack is non-empty: append current node as a child of stack[-1]
      - Else: append to roots
      - If not section.is_table (table leaves do not own children): push current node onto stack
    - After building all nodes, derive page_end for every node:
      - Walk the full node list in original order; for each node, page_end = next node's page_start - 1 if the next node is at same or shallower level; otherwise use page_count
      - For the absolute last section: page_end = page_count
      - Table leaves: page_end = page_start (single page)
    - Return roots

    No-sections fallback: when parsed_doc.sections is empty, return [TreeNode(node_id="§1", title=parsed_doc.metadata.get("title","§1"), summary=parsed_doc.metadata.get("title","§1"), page_start=1, page_end=parsed_doc.metadata.get("page_count",1), level=1, section_path="§1")]

    Main function tree_index(parsed_artifact_id: str, source_id: str, parsed_doc: ParsedDoc, *, settings: Settings | None = None) -> dict[str, Any]:
    1. Load settings via get_settings() if None
    2. Determine mode from settings.tree.mode
    3. Open a registry session and load the parsed artifact's content_hash (Finding 2 per RESEARCH.md)
    4. Compute content_hash per D-06
    5. Check no-op: get_artifact_by_hash(session, content_hash, "tree_index") — return cached result if hit
    6. Build deterministic tree: call _build_deterministic_tree
    7. If mode == "llm": check budget and either return skipped_budget_exceeded or call litellm.completion per node to replace summaries; validate each response with NodeSummaryResult.model_validate_json(); bound the node text excerpt fed to the prompt to a fixed character limit; record_llm_spend after all nodes processed; on any LLM exception, log warning and keep the deterministic summary for that node (never raise)
    8. Serialize TreeIndex to JSON via orjson.dumps() (not stdlib json)
    9. Resolve domain via get_domain_for_source; construct storage key per D-07
    10. put_object with tags; register via create_tree_index_artifact
    11. Return {"artifact_id": art.id, "cached": False, "status": "ok"}

    Imports to use: from __future__ import annotations; hashlib; structlog; orjson; litellm; tenacity (for @retry on LLM call); from knowledge_lake.config.settings import Settings, get_settings; from knowledge_lake.plugins.protocols import ParsedDoc, Section, TreeNode, TreeIndex; from knowledge_lake.registry import repo as registry_repo; from knowledge_lake.registry.db import get_session; from knowledge_lake.storage.s3 import _UNCLASSIFIED_DOMAIN, StorageBackend; from pydantic import BaseModel (for NodeSummaryResult).
