<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| parse()/clean() → registry (get_domain_for_source) | Domain value comes from Source.config in the DB; caller resolves and passes down to put_object — storage layer never calls the registry |
| pipeline → S3 (put_object with tags) | Tags constructed from DB values (domain, source_name); tag values truncated to 256 chars in _format_tags (implemented in Plan 03) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-09-01 | Denial of Service | tag values (source_name from registry) | medium | mitigate | _format_tags truncation to 256 chars (implemented Plan 03); source_name from registry is already a short label |
| T-09-03 | Tampering | session boundary for get_domain_for_source | medium | mitigate | Both calls placed inside active with get_session() block (Pitfall 3 avoidance); DetachedInstanceError cannot occur |
| T-09-04 | Information Disclosure | source_name fallback to "unknown" | low | accept | If get_source returns None, tag carries "unknown" rather than omitting source_name; registry remains source of truth |
</threat_model>
