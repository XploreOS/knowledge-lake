| gold export put_object → S3 | Tags on gold objects are 3-key (domain, format, artifact_type); source_name deliberately omitted for multi-source exports |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-09-01 | Denial of Service | tag values in gold export | low | mitigate | _format_tags truncation (Plan 03) applies; domain and format are short strings well under 256 chars |
| T-09-03 | Tampering | domain segment in gold keys | medium | accept | domain kwarg is passed by the calling pipeline or CLI (not direct user input to key construction); "_unclassified" guard prevents empty/None segments |
| T-09-06 | Information Disclosure | gold key exposes domain name in S3 path | low | accept | domain is a label like "healthcare" already stored in Source.config; not a secret; S3 bucket access is already controlled by IAM/MinIO ACLs |
</threat_model>
