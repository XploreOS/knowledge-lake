Imports: from __future__ import annotations, re, structlog, from knowledge_lake.config.settings import Settings and get_settings, from knowledge_lake.pipeline.search import search, from knowledge_lake.pipeline.tree_search import tree_search, from knowledge_lake.plugins.protocols import Hit. Set log = structlog.get_logger(__name__).

Module-level _TREE_TRIGGERS: a list of (category_name, compiled_re_pattern) tuples. Three categories per D-04 (case-insensitive):
- "section_page_ref": triggers on section+digit, section-mark, page(s)+digit, chapter+digit
- "comparison_multihop": triggers on compare, difference between, vs/versus, how does X affect/relate to/impact
- "structural_breadth": triggers on outline of, table of contents, all sections, summarize the/all

All patterns MUST be linear (no nested quantifiers) to prevent ReDoS. No user input is interpolated into patterns.

classify_route(query: str) -> tuple[str, str]: iterate _TREE_TRIGGERS, return ("tree", category) on first match, else ("chunk", "no_match"). This is the deterministic-first classifier (D-04, ROUTE-02, ROUTE-03).

routed_search() signature: (query: str, *, route: str | None = None, collection: str = "klake_chunks", top_k: int = 5, mode: str | None = None, settings: Settings | None = None, domain: str | None = None, document_type: str | None = None, min_quality_score: float | None = None, source_name: str | None = None, format: str | None = None, tags: list[str] | None = None, source_id: str | None = None) -> list[Hit].

Implementation:
1. Empty-query guard: if not query.strip(), log warning, return [].
2. Resolve settings: s = settings or get_settings().
3. Resolve route: effective_route = route or s.router.default_route (Pattern 1 per-call override).
4. If effective_route == "auto": call classify_route(query) to get (decided, category). If decided == "tree": call tree_search(query, collection=collection, top_k=top_k, mode=mode, settings=s) — note: tree_search accepts NO filter kwargs (top_k/mode/max_docs/collection/settings only). If result is non-empty, log and return. If empty, log with fallback=True, call search() with all filter kwargs, return result (even if empty — D-05, Pitfall 4). If decided == "chunk": log trigger="no_match", call search() with all filter kwargs.
5. If effective_route in ("tree", "two_stage"): log trigger="operator_override", call tree_search(), return result directly (NO fallback — D-05). tree and two_stage are aliases (D-01).
6. If effective_route == "chunk": log trigger="operator_override", call search() with all filter kwargs.

Structlog event (D-06): one log.info("route.dispatch", route=effective_route_used, trigger=category_or_override, fallback=bool) per call. The "route" in the event is the actual path taken (chunk or tree), not the input value.

MUST be synchronous (plain def, not async) — tree_search() raises RuntimeError if called from within a running event loop (Pitfall 2).
