## Phase Goal (slice framing)

**As a** knowledge-lake operator, **I want to** export my curated corpus, chunk citations, and generated datasets to standard Parquet/JSONL formats that any downstream tool can consume, **so that** the RAG corpus is queryable via plain SQL (DuckDB), the pretraining corpus is a clean JSONL text stream, and the fine-tuning dataset is in a standard chat-messages format — without hand-building a format reader for this framework.

<objective>
Implement EXPORT-01 (RAG corpus -> Parquet, queryable via DuckDB), EXPORT-02 (pretraining corpus -> JSONL from curated_document text), and EXPORT-03 (fine-tuning dataset -> OpenAI chat-messages JSONL from dataset_examples) as the final vertical slice of Phase 5, closing the loop from Plan 05-01's curated_document + Plan 05-02's dataset_examples into standard gold-zone deliverables. Per D-09, all three write to a new gold zone in the EXISTING `StorageBackend` (raw -> bronze -> silver -> gold zone progression) — never a new storage backend.

Purpose: Turn this phase's curated + generated artifacts into the AI-ready deliverables the whole framework exists to produce (PROJECT.md Core Value).
Output: `src/knowledge_lake/pipeline/export.py` (`export_rag_corpus()`, `export_pretrain_corpus()`, `export_finetune_dataset()`, `verify_export()`), `ExportSettings`, `klake export` CLI command, `POST /exports` API endpoint, export Dagster assets.
</objective>
