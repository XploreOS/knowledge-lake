<objective>
Deliver the index/search vertical slice: Qdrant alias-based collection management with zero-downtime reindex, an extended chunk payload carrying domain/document_type/keywords/quality_score from the sibling enrichment, and filterable, backward-compatible semantic search across CLI/API.

Purpose: INDEX-01..03 make the corpus's enrichment metadata actually queryable — a search result can be filtered by domain/document_type/quality_score, and reindexing (e.g. after a payload schema change or embedding-model swap) never takes search down, closing STATE.md's second Phase-4 blocker (Qdrant collection aliasing patterns).

Output: `QdrantVectorStore.ensure_aliased_collection/reindex/copy_all_points`, extended `pipeline/index.py` payload join, extended `pipeline/search.py` filters, `klake search` flags + `klake reindex` command, `GET /search` filter params + `POST /reindex`.
</objective>

## Phase Goal (slice framing)

**As a** knowledge-lake operator, **I want to** search my indexed chunks with citation, domain, document-type, and quality-score filters, and reindex the whole collection with zero search downtime, **so that** I can find and trust relevant passages today and safely evolve the index (new payload fields, a new embedding model) tomorrow without ever taking search offline.
