<objective>
Wire DataTrove's `FineWebQualityFilter` and Phase 19's pure predicate module (`pipeline/quality/`) into `chunk()` as a composite substance gate — rejecting (enforce mode) or flagging (report mode) garbage chunks before they are persisted, embedded, or indexed. Also folds `filter_config_version` into the existing per-chunk WR-05 content hash so a threshold change forces re-processing on the next run (PIPE-01).

Purpose: This is the load-bearing implementation for QUAL-02 (chunk-scope FineWebQualityFilter), QUAL-03 (composite substance gate with enforce/report modes and `is_table` exemption), and PIPE-01 (filter-config cache versioning) — the core mechanism the milestone's "<5% garbage chunks" success criterion depends on. Both the Dagster asset and CLI/API path call `chunk()` directly, so this single change gives automatic parity across both pipeline entry points (D-01).

Output: `ChunkQualitySettings` (new settings model), `chunk()` extended with a `domain_filters` parameter and an internal substance gate, a new `tests/unit/test_chunk_substance_gate.py`, and PIPE-01 cache-versioning assertions added to `tests/unit/test_chunk_storage.py`.
</objective>
