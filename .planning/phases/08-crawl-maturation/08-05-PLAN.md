<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Crawled HTML content → linked URLs | HTML from untrusted sites may contain crafted hrefs pointing to internal services (SSRF) |
| Linked PDF/DOCX URLs → ingest_url path | Every followed URL must be SSRF-validated before any HTTP is issued |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-08-05-01 | Elevation of Privilege | SSRF via linked document URL (D-21) | high | mitigate | validate_public_url called on every followed link before ingest_url, every time, no exceptions — defense in depth even though ingest_url also calls it internally (RESEARCH.md Security Invariant) |
| T-08-05-02 | Denial of Service | unbounded linked-doc frontier | medium | mitigate | MAX_LINKED_DOCS_PER_PAGE = 10 per page; _seen_urls dedup prevents re-following; linked docs NOT re-crawled for further links (RESEARCH.md) |
| T-08-05-03 | Elevation of Privilege | SSRF via redirect in followed link | medium | mitigate | ingest_url's _fetch_with_retry validates redirect hops (ingest.py:191); outer validate_public_url call provides first-hop guard |
| T-08-05-04 | Denial of Service | event loop blocking from sync ingest_url | medium | mitigate | run_in_executor wraps every ingest_url call (Pitfall 3 mitigation) — offloads httpx.Client to thread pool |
</threat_model>
