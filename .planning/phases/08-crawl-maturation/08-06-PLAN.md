## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| API caller → POST /crawl-all | External callers can trigger batch crawl of all registered sources |
| CLI operator → klake crawl-all | Operator-initiated; same trust level as klake crawl |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-08-06-01 | Denial of Service | POST /crawl-all triggers sequential crawl of all sources | medium | mitigate | Sequential loop (D-06); per-host adaptive backoff from Plan 2; crawl-all is no more aggressive than N consecutive single-source crawls; no concurrency amplification |
| T-08-06-02 | Tampering | domain query param in POST /crawl-all | low | accept | domain is passed as a filter to list_sources_for_crawl_all; Python-side string equality check; no SQL injection risk (SQLAlchemy ORM, Python-side filter) |
| T-08-06-03 | Information Disclosure | error details in CrawlAllSourceResult.error | low | accept | Error messages are internal pipeline exceptions; no user data in error strings; consistent with existing /crawl-jobs error exposure |
</threat_model>
