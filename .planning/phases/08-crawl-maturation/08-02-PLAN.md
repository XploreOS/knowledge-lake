<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Source.config JSON → rate-limit logic | Attacker-influenced source config could set rate_limit_rps = 0 to trigger divide-by-zero |
| HTTP response status code → backoff logic | Attacker-controlled server returns 429 to force aggressive backoff (DoS on the crawler) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-08-02-01 | Tampering | resolve_delay rate_limit_rps path | medium | mitigate | Guard: if rps > 0 else global_default (per D-03/Pattern 2); zero/negative rps falls through to global default |
| T-08-02-02 | Denial of Service | PerHostLimiter backoff_extra | low | accept | MAX_BACKOFF_SECONDS caps the sleep at 60s; per-host, not global; attacker must own the target server to trigger |
| T-08-02-03 | Tampering | CrawlPageResult http_status_code | low | accept | Field is set from adapter-side network response; it informs backoff decisions but does not cross a trust boundary into auth/storage |
</threat_model>
