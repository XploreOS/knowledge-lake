## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → search_endpoint | source_name, format, source_id, tags arrive as FastAPI Query params; Pydantic validates types before handler runs |
| CLI user → cmd_search | flags arrive as typer-parsed Optional[str]/list[str]; typer validates types before handler runs |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-07-04-01 | Denial of Service | search_endpoint tags Query param | medium | mitigate | Per RESEARCH.md security domain: add `max_length=64` on the tags Query param to bound per-element string length. The existing top_k [1,100] bound prevents result-set explosion. Addressed in Task 1 Step 2(a). |
| T-07-04-02 | Information Disclosure | SearchHit new fields | low | accept | source_url, organization, tags are already stored in the public Source registry; exposing them in search results adds no new disclosure surface. source_id is a registry-internal ID — not a secret. |
| T-07-04-03 | Tampering | CLI/API filter params → Qdrant | low | accept | source_name, format, source_id are passed as FieldCondition match values (parameterized, not concatenated). Qdrant client handles escaping. Path traversal is moot for filter strings (RESEARCH.md security domain). |
| T-07-04-04 | Tampering | collection name | low | accept | Existing _COLLECTION_NAME_RE guard in app.py still validates collection before reaching search(). No change needed. |
</threat_model>
