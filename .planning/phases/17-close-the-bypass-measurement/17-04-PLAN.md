<objective>
Build the MEAS-01 quality-audit harness: a re-runnable, reproducible per-source garbage-rate table that re-runs the real `parse`/`clean` pipeline (D-07 — no separate frozen classifier) against every source in a domain and surfaces QUAL-04's rejection counts. Ship as a new `pipeline/quality_audit.py` module plus a `klake quality-audit` Typer command.

Per D-06, the audit's `domain` scope defaults to `"healthcare"` and queries whatever sources are actually registered under `Source.domain == "healthcare"` at run time — matching the original audit's scope intent without hardcoding its historical row count (Pitfall 4). Per D-08, the column set (`source_name, sections_considered, sections_kept, sections_rejected, documents_errored, garbage_rate`) satisfies MEAS-01's minimum bar (total sections, kept, rejected, rejection reasons, garbage rate) plus one Claude-discretion addition (`documents_errored`) for QUAL-05-spirited error visibility.

Purpose: This is the measurement infrastructure every subsequent v2.6 phase (19, 20, 21) needs a frozen baseline from. Per RESEARCH.md Pitfall 5, this phase's own audit output will legitimately show near-zero rejections (Phase 17 does not drop any sections by design) — this plan builds the harness and establishes the Phase-17 baseline; meaningful garbage-rate reduction is Phase 19/20's job, measured against this same harness later.
Output: `run_quality_audit()` (pipeline function, CLI-agnostic per D-03's "one function, many callers" precedent) and the `klake quality-audit` CLI command.
</objective>
