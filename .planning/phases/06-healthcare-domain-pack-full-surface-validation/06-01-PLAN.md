<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem → DomainLoader | domain name parameter controls which path is read; domain YAML/Python loaded from filesystem |
| domain pack → Python runtime | validators/validate.py is loaded dynamically via importlib.util |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-01 | Tampering | DomainLoader.from_name() path construction | high | mitigate | Validate domain name against r"^[a-zA-Z][a-zA-Z0-9_-]{0,63}$" before constructing path; raise ValueError on mismatch (path traversal guard) |
| T-06-02 | Elevation of Privilege | importlib dynamic load of validators/validate.py | medium | accept | Domain packs are operator-controlled artifacts placed by the developer; document that arbitrary Python execution is the intended behavior; validate domain_dir is inside KLAKE_DOMAINS_ROOT before loading |
| T-06-03 | Information Disclosure | PHI heuristic matched text in logs | high | mitigate | validate.py logs only "phi_gate_triggered=True" — never logs the matched text span |
| T-06-04 | Tampering | yaml.load() code injection via YAML tags | medium | mitigate | Use yaml.safe_load() exclusively for all YAML parsing; never call yaml.load() |
| T-06-05 | Tampering | Jinja2 autoescape corruption of clinical codes | low | mitigate | jinja2.Environment(autoescape=False) for all prompt templates |
| T-06-SC | Tampering | npm/pip/cargo installs | high | mitigate | pyyaml and jinja2 are existing transitive deps with verified versions; slopcheck complete (see RESEARCH.md Package Legitimacy Audit) |
</threat_model>
