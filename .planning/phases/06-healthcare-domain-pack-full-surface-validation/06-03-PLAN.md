| HTTP client → API /domains/{name}/sources | domain name in URL path segment controls filesystem path construction |
| CLI user → klake init --domain | domain name argument controls filesystem path construction |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-08 | Tampering | /domains/load body.name path traversal | high | mitigate | Validate name against r"^[a-zA-Z][a-zA-Z0-9_-]{0,63}$" pattern in DomainLoadRequest Pydantic schema AND in the endpoint before DomainLoader.from_name(); return 422 on invalid name |
| T-06-09 | Tampering | /domains/{name}/sources path traversal | high | mitigate | Same regex validation on name path parameter; raise HTTPException(422) if pattern does not match before calling DomainLoader |
| T-06-10 | Information Disclosure | GET /sources returning internal metadata | low | accept | Sources are operator-registered registry entries, not user secrets; no credentials or keys stored in Source rows |
| T-06-11 | Tampering | SQL injection via domain/source_id query params | medium | mitigate | All queries use SQLAlchemy ORM select() with bound parameters — never string-formatted SQL |
</threat_model>
