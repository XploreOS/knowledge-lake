## Dagster deferral note (Open Question #3 — surfaced, not dropped)

Per CLAUDE.md "Dagster from day 1", crawl is implemented as plain functions plus durable `crawl_states` (any driver — CLI, API, or a future Dagster asset — can start/resume a crawl), consistent with the Phase-1 D-01/D-02 pattern where Dagster assets wrap the same plain pipeline functions. A thin `crawl_source` Dagster asset/sensor for long-running observability is **explicitly deferred to Phase 6 / IFACE-03** (research Open Question #3). This is a recorded decision, not a silent drop; resume works today without Dagster because state is persisted in `crawl_states`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| crawler → followed link | Every crawled/followed URL is attacker-influenceable (a page can link to IMDS/RFC-1918) |
| target host → adapter | Response bytes/markdown from an untrusted host cross into storage |
| CLI/API → crawl_source | Operator-supplied source_url + crawler override |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-09 | Information Disclosure (SSRF via crawled link) | pipeline/crawl.py + adapter | high | mitigate | Call validate_public_url on EVERY URL before fetch (Pitfall 2 — the existing guard covered only ingest_url); default same-registrable-domain scope. |
| T-02-10 | Repudiation / legal (robots bypass) | orchestrator + Crawl4AI check_robots_txt | high | mitigate | Native robots via check_robots_txt=True; disallowed → crawl_states.status="robots_blocked", no artifact (D-13). |
| T-02-11 | DoS (self-inflicted on target host) | ratelimit in orchestrator | high | mitigate | Per-host three-tier delay awaited before each fetch; max_pages/same_domain bound scope. |
| T-02-12 | DoS (oversized response) | adapter | medium | mitigate | 50 MB size posture on returned bytes; per-page handling. |
| T-02-13 | Tampering (crawler override injection) | CLI/API crawler param | low | mitigate | Validate crawler value against the known crawler set (ASVS V5). |
| T-02-SC | Tampering (supply chain) | crawl4ai==0.9.0 | low | accept | Approved in RESEARCH Package Legitimacy Audit (project-designated primary crawler). No [ASSUMED]/[SUS]/[SLOP]. |
</threat_model>
