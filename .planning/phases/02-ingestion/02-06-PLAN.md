Deliver source discovery: a swappable `DiscoveryPlugin` protocol with SearXNG as the first implementation (JSON API), a discovery pipeline that auto-registers each result as a minimal `source_type='discovered'` source (URL + title only) after SSRF-validating and URL-deduping it, plus the SearXNG Docker service wired into the compose stack with the JSON output format enabled. ## Phase Goal (slice framing)

**As a** knowledge-lake operator, **I want to** run a search query and have candidate public sources auto-registered in the lake for review, **so that** I can grow the source registry by discovery instead of hand-entering every URL — with SSRF-safe, deduped, minimally-described entries I can later assign a domain and crawl.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SearXNG result → registry | Result URLs are attacker-influenceable (a poisoned result could point at IMDS/RFC-1918) |
| operator query → SearXNG | Query string crosses into an HTTP request to the internal SearXNG service |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-22 | Information Disclosure (SSRF via discovered URL) | pipeline/discover.py | high | mitigate | validate_public_url on every result URL before auto-register/crawl (Pitfall 2); skip+log on failure. |
| T-02-22b | Information Disclosure (redirect-hop / DNS-rebinding SSRF) | discover fetch path | high | mitigate | Any subsequent fetch of a discovered URL reuses `_fetch_with_retry` (from 02-01) which validates each redirect hop. A discovered URL that 302-redirects to a private/link-local IP is rejected before the private host is contacted. |
| T-02-23 | Tampering / SSRF (query injection into SearXNG) | SearXNGDiscovery | medium | mitigate | Pass q as an httpx params value (httpx URL-encodes); never string-format into the URL; treat searxng_url as trusted internal config only. |
| T-02-24 | Tampering (403 JSON disabled → silent break) | infra/searxng/settings.yml | low | mitigate | Enable formats: [html, json]; raise a clear error on 403 (Pitfall 3); settings.yml assertion test. |
| T-02-25 | DoS (unbounded result registration) | discover_sources / API | low | mitigate | Bound limit with pydantic Field(ge=1, le=...); URL-first dedup prevents duplicate rows. |
| T-02-SC | Tampering (supply chain) | searxng/searxng Docker image | low | accept | Established self-hosted meta-search (project-designated, no API keys). No new PyPI deps (httpx already pinned). |
</threat_model>
