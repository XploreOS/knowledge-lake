<objective>
Deliver the first Phase-2 user capability and the shared ingestion foundation: register sources, download single URLs, and upload local files — all idempotent (URL-first, hash-second dedup) with full provenance. This plan also extracts the existing single-purpose SSRF guard into a shared `validate_public_url()` that every later crawler/discovery plan re-uses, and adds the conservative D-06 URL normalizer plus the `sources.normalized_url` column that URL-first dedup depends on.

Purpose: INGEST-01/02/03/08 are the base of every downstream slice; the shared SSRF guard and URL normalizer are prerequisites for crawling and discovery.
Output: `klake add-source`, `klake upload`, dedup-aware `klake ingest-url`; `POST /sources`, `POST /uploads`; Alembic 0002; `validate_public_url`, `normalize_url`, `register_source`.
</objective>

## Phase Goal (user story)

**As a** knowledge-lake operator, **I want to** register, download, and upload public resources into the lake with provenance and dedup, **so that** any single public source becomes a traceable raw-zone artifact I can build on — without ever creating duplicate rows for the same URL or content.

> This plan is the Wave-1 foundation slice. It delivers a real end-to-end capability (register/download/upload) AND the shared primitives (`validate_public_url`, `normalize_url`, `sources.normalized_url`) that the crawler and discovery slices depend on.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI/API caller → pipeline | Operator-supplied URL, file path, name, domain, license cross into ingest logic |
| pipeline → remote host | `ingest_url` fetches an operator-supplied URL (attacker-influenceable if the operator was tricked) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-01 | Information Disclosure (SSRF) | pipeline/ingest.py fetch | high | mitigate | Extract `validate_public_url()` (https-only + getaddrinfo private/IMDS block, verbatim from `_validate_url_scheme`); call it before every fetch. This is the shared guard consumed by 02-02..02-06. |
| T-02-02 | Tampering (injection) | repo lookups on normalized_url / hash | medium | mitigate | ORM-only `select().where()` for get_source_by_normalized_url and get_raw_artifact_for_source (no raw SQL, T-01-03). |
| T-02-03 | DoS (oversized download) | ingest_url | medium | mitigate | Reuse existing 50 MB `MAX_DOWNLOAD_BYTES` streaming cap. |
| T-02-04 | Tampering (API input) | POST /sources, /uploads | medium | mitigate | Pydantic-validate body fields; reject malformed URL/name at the boundary (ASVS V5), matching `_COLLECTION_NAME_RE` posture. |
| T-02-SC | Tampering (supply chain) | dev dep `hypothesis` | low | accept | Established, high-download OSS test lib; no runtime/prod surface. No [ASSUMED]/[SUS] packages in this plan (RESEARCH Package Legitimacy Audit). |
</threat_model>
