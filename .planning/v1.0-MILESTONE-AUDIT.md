### Gaps (2 — non-blocking)

**GAP 1 — DOMAIN-03: Domain prompt override not wired at runtime (minor)**

The `domain_system_prompt` kwarg exists in `pipeline/enrich.py::enrich_document()` (line 213) and `_build_enrichment_prompt()` (line 129) correctly uses it when provided. The `DomainLoader.render_prompt()` method exists in `domains/loader.py` (line 127). The `enrich.j2` template exists in `domains/healthcare/prompts/`. However, no caller passes the rendered prompt:
- `cli/app.py::cmd_enrich()` — calls `enrich_fn()` without `domain_system_prompt`
- `api/app.py::enrich_endpoint()` — calls `enrich_document()` without `domain_system_prompt`
- `dagster_defs/assets.py::enrich_document` asset — calls `enrich_fn()` without `domain_system_prompt`

**Effect:** Healthcare E2E runs use the generic enrichment prompt, not the clinical-entity-extraction prompt in `enrich.j2`. The pipeline completes; clinical code extraction quality is degraded.

**Fix:** In each caller, check `settings.domain.domain_name`; if set, call `DomainLoader.from_name(name).render_prompt("enrich.j2", ...)` and pass the result as `domain_system_prompt`. ~15 lines across 3 files.

---

**GAP 2 — IFACE-02: `POST /dedupe` endpoint missing (minor)**

`klake dedupe` calls `pipeline.curate.batch_dedup_corpus()`. `api/app.py` has no corresponding endpoint. `POST /curate` calls per-document `curate_document()`, not the batch-corpus function.

**Effect:** The FastAPI surface does not expose batch corpus deduplication. IFACE-02's "curation" endpoint group is partially covered (per-document curation only).

**Fix:** Add `POST /curate/dedupe` or `POST /dedupe` endpoint calling `batch_dedup_corpus()`. ~20 lines.
